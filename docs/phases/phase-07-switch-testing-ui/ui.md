## Sections

### 1. Profile Selector

- Radio group or card group: ATM | POS | Reversal | Fraud | Custom
- Loaded from `GET /api/v1/testing/profiles` on mount
- Selecting a profile pre-fills the field editor with the profile's default values
- Displays the profile description below the selection
- Selecting **Custom** clears all fields to empty

### 2. Field Editor

Editable form showing the ISO 8583 fields used by the switch.

| Label | ISO Field | Notes |
|---|---|---|
| MTI | — | e.g. `0200`, `0420` |
| PAN | F2 | Masked in display (show last 4) |
| Processing Code | F3 | 6-digit code |
| Amount | F4 | 12-digit zero-padded (minor units) |
| STAN | F11 | Auto-generated by service if left blank |
| POS Entry Mode | F22 | 3-digit |
| RRN | F37 | Auto-generated by service if left blank |
| Terminal ID | F41 | 8-char |

- Fields pre-filled by profile selection are editable (operator can override any value)
- Fields marked "auto" are left blank by default; the service fills them before sending
- Amount field has a human-readable helper: `000000010000` → displayed as `100.00`
- Send button is disabled while a request is in-flight (loading spinner)

### 3. Response Panel

Displayed after each send. Cleared by **Clear** button.

| Element | Content |
|---|---|
| RC badge | Green ✅ for `00`, red ❌ for any other code + code label |
| MTI | Response MTI (e.g. `0210`) |
| STAN | Field 11 from response |
| RRN | Field 37 from response |
| Elapsed | `elapsed_ms` from API response |
| Raw Fields table | All fields returned in `response.fields` |
| Copy button | Copies full JSON response to clipboard |

### 4. History Table

- Loaded from `GET /api/v1/testing/history` after each send
- Columns: `#` | Sent At | Profile | RC | STAN | RRN | Elapsed
- RC column colour-coded: green for `00`, red otherwise
- Clicking a row re-populates the field editor with the original request fields
- Max 20 rows shown; resets on service restart

## API Service (`frontend/src/services/api.js`)

Three new functions to add:

```js
// Get available profiles
export const getTestingProfiles = () =>
  api.get('/testing/profiles');

// Send a test ISO transaction
export const sendTestTransaction = (payload) =>
  api.post('/testing/send', payload);

// Get test history
export const getTestingHistory = (limit = 20) =>
  api.get('/testing/history', { params: { limit } });
```

```js
    '4':  '000000999999',
    '22': '029',
    '41': 'TERM9999',

  sending: false,          // loading state
  response: null,          // last API response object

```

## Error Handling

| Scenario | UI Behaviour |
|---|---|
| Switch unreachable (503) | Red alert banner: "Switch is not reachable. Is the switch container running?" |
| Validation error (400) | Field-level error message below the offending input |
| Network error | Toast notification with error message |
| RC ≠ 00 | Response panel shows red RC badge with code + description |
